TRACE：面向大语言模型智能体的开源分层记忆系统在EventQA上达到82.5% F1值

新闻时间：2026-07-06T14:35:56+00:00 · 来源：reddit

一种名为TRACE的新型开源记忆架构正成为大语言模型（LLM）智能体传统扁平化检索增强生成（RAG）系统的有力替代方案。与将对话历史简单切片或无序存储的方法不同，TRACE采用分层方式组织记忆——动态构建一棵主题树，每个分支代表一个语义连贯的主题，并附带摘要和时间关联标记。这一设计灵感部分源自心理学中的多重痕迹理论，该理论认为每次经验都会在大脑中形成独特、多属性组合的记忆痕迹，而非单一复制品。

该系统在MemoryAgentBench的EventQA任务上接受了严格评测——这是ICLR 2026会议提出的一项新基准，旨在检验智能体对小说中时序事件链的理解与推理能力。EventQA采用“一次注入、多次查询”范式：一段长篇叙事文本被逐步输入（多轮交互），随后用多个问题探测模型对“接下来发生什么”的预测准确性。这种设置不仅考验关键词匹配能力，更强调长期记忆保持、事件连贯性追踪与因果推理。

在开源权重模型GPT-OSS-20B（210亿参数混合专家模型，每步激活36亿参数）上，TRACE取得了82.5%的F1分数，大幅超越现有基线：Mem0（使用GPT-4o-mini官方报告值）为37.5%，MemGPT/Letta为26.2%。需注意，这些对比并非完全同构——因模型架构与API调用方式差异显著，但性能鸿沟依然突出。作者坦诚说明了实验限制：尝试将Mem0适配至GPT-OSS-20B时遭遇JSON解析失败（该问题在Gemini、Mistral等模型中亦普遍存在），而Letta依赖完整服务端部署，故未纳入实测。

TRACE已发布为PyPI包（pip install trace-memory），并公开全部JSON日志以保障可复现性。其模块化设计支持无缝接入现有智能体框架；主题树结构则便于人工调试、剪枝及交互式记忆导航。尽管可扩展性与最优树深度控制仍是待解问题（Reddit讨论区中用户正热烈探讨），该项目已为开源权重智能体记忆评估树立了新标杆。
