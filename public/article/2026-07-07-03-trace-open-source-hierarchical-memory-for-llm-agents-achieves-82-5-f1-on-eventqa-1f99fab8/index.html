TRACE：面向大语言模型智能体的开源分层记忆系统，在EventQA任务上达到82.5% F1值

新闻时间: 2026-07-07T02:27:41.799274+00:00 · 来源: reddit

一款名为TRACE的全新开源记忆架构正在重新定义大语言模型（LLM）智能体如何存储与检索对话知识。与传统检索增强生成（RAG）系统不同——后者将对话历史扁平化为彼此割裂、上下文无关的文本片段——TRACE采用分层方式，将交互组织成主题树结构。每个节点代表一个连贯的主题或事件，分支承载子主题，摘要则提炼关键信息。这种结构模拟人类记忆的组织逻辑，能更精准地回溯长程对话中的时间顺序与因果关系。

该系统在MemoryAgentBench的EventQA任务上接受了严格评测——这是ICLR 2026会议提出的基准测试，专门评估智能体对小说等叙事文本中事件链的理解与“下一步发生什么”的预测能力。使用开源权重模型gpt-oss-20B（210亿参数，实际激活36亿），TRACE取得了82.5%的F1分数，大幅超越此前主流开源记忆框架：Mem0（使用GPT-4o-mini）得分为37.5%，MemGPT/Letta仅为26.2%。即使换用更大的gpt-oss-120B模型，TRACE仅小幅提升至83.8%，表明其在小模型上已具备极强的记忆建模能力。

值得注意的是，评测明确承认了方法论限制：这是在本地运行的开源模型与闭源模型公开结果之间的跨模型对比，并非同构基座模型的公平比较。作者曾尝试将Mem0迁移至gpt-oss-20B，但在事实抽取环节因JSON输出格式不稳定而失败——该问题并非gpt-oss特有，Gemini和Mistral等模型也存在类似解析缺陷。Letta则因需完整服务端部署被跳过。为保障可复现性，全部原始推理日志均已开源至GitHub仓库。

TRACE以PyPI包形式发布（pip install trace-memory），专为开发者集成至智能体工作流设计。其分层结构直击扁平化记忆系统的根本缺陷：难以处理动态演化的主题、交叉叙事及时间依赖关系。随着LLM智能体向长周期、多步骤推理场景（如客服对话、科研辅助、互动叙事）演进，结构化记忆正成为不可或缺的基础设施。TRACE不仅带来性能突破，更提供了一个可审计、可扩展、可复现的记忆研究新基线。
