uv 0.11.28 发布：强化 ZIP 解析安全性并升级 GraalPy 至 25.1.3

新闻时间: 2026-07-08T04:29:38.425107+00:00 · 来源: github

Rust 编写的 Python 工具链项目 uv 正式发布 0.11.28 版本，标志着其在安全加固与开发者体验优化两方面取得重要进展。该版本于 2026 年 7 月 7 日发布，核心安全更新在于将底层 ZIP 处理库 astral-async-zip 升级至 v0.0.20。ZIP 格式长期是软件供应链攻击的高危入口点；此次升级引入 15 项针对性改进，专门防范“解析器差异”（parser differential）——即下载器、解压器与安装器对同一 ZIP 文件产生不一致解释，从而绕过校验、执行恶意代码。

除安全外，本次发布还将内嵌的 GraalPy 运行时升级至 25.1.3 版本。GraalPy 是基于 GraalVM 的高性能 Python 3 实现，支持 JIT 编译，在数据科学与机器学习场景中表现优异，尤其对 PyTorch 和 SciPy 兼容性良好。此次升级确保 uv 能持续适配最新 GraalVM 生态演进，并体现其支持多 Python 运行时（不止 CPython）的战略方向，提升跨语言与云原生环境下的灵活性与性能。

面向开发者的体验优化聚焦于错误可读性与一致性：多行错误原因现在保留原始缩进；错误根源链（cause chain）完整呈现；uv build 和工具依赖报错均采用统一渲染格式，并严格遵循 -q（静默）和 -qq（超静默）标志。锁文件更新提示也从警告迁移为上下文提示，既降低干扰又不失指导性。这些改进显著降低了调试与 CI 故障排查的认知负担。

性能优化则体现在 22 处内存分配规避措施中，覆盖依赖解析、URL 处理、平台标识、元数据格式化等环节。虽单点影响微小，但叠加后可在大型依赖图或资源受限环境（如 CI 构建节点）中带来可观延迟下降。尤为值得注意的是，uv pip install 现仅对实际安装的包编译字节码，避免了对缓存或解析阶段包的冗余处理。整体而言，本次更新进一步巩固了 uv 作为快速、安全、易用的现代 Python 工具链替代方案的地位。
