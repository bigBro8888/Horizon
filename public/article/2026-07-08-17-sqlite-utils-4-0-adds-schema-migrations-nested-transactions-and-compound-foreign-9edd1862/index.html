sqlite-utils 4.0 发布：支持数据库迁移、嵌套事务与复合外键

新闻时间: 2026-07-08T04:29:37.384585+00:00 · 来源: rss

西蒙·威利森（Simon Willison）正式发布了 sqlite-utils 4.0 —— 这款广受欢迎的 SQLite Python 库与命令行工具五年来的首个重大版本更新。作为该项目第 124 次发布，也是继 2020 年 11 月 v3.0 之后的首次主版本升级，4.0 引入了三项核心能力：内置数据库模式迁移（schema migrations）、通过 db.atomic() 实现的嵌套事务，以及对复合外键（compound foreign keys）的原生支持。这些特性共同填补了轻量级 SQLite 工具链中长期存在的关键空白，尤其适用于构建数据驱动原型、个人分析系统或小型生产应用的开发者。

全新的迁移系统允许用户用纯 Python 函数定义可版本化、渐进式的数据库结构变更。每个迁移函数接收一个 Database 实例，并可执行任意操作：建表、加列、调用 table.transform() 重构表结构，或插入初始数据。尤为关键的是，table.transform() 严格遵循 SQLite 官方推荐的复杂变更模式：先创建新结构的临时表，安全迁移数据，再原子性地删除旧表并重命名新表——从而绕过 SQLite 原生 ALTER TABLE 的功能限制。所有迁移记录自动存入 _migrations 系统表，确保幂等执行，杜绝重复应用。

执行迁移极为简单：运行 uvx sqlite-utils migrate data.db migrations.py 即可按序应用所有待处理函数。开发者还可使用 --list 列出迁移项、--verbose 查看迁移前后的完整模式差异，或用 --stop-before 在指定步骤前终止执行。该实现源自威利森早期独立项目 sqlite-migrate，经多年实际验证后正式整合进主库，体现了成熟度与可靠性。与此同时，db.atomic() 支持真正的嵌套事务：内层 atomic() 块可独立回滚，不影响外层事务状态——显著优于 SQLite 默认的扁平事务模型。

复合外键支持使 sqlite-utils 更贴近 SQLite 的完整 SQL 标准：开发者现在能声明引用多个列的外键（例如 (order_id, item_seq) → (orders.id, order_items.seq)），精准建模多对多关系与联合主键。尽管 SQLite 默认禁用外键约束（需显式执行 PRAGMA foreign_keys = ON），sqlite-utils 4.0 已提供简洁的 Python API 与 CLI 接口，确保此类约束被正确定义与管理。这三项增强合力将 sqlite-utils 推向新高度——它不再仅是便捷工具，更成为可演进、可信赖的 SQLite 生产级基础设施。
