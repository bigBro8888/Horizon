自托管式NATS聊天平台Chatto正式开源

新闻时间: 2026-07-09T05:04:10.099604+00:00 · 来源: hackernews

Chatto——一款注重隐私、支持自托管的实时聊天平台——现已正式开源。该项目由知名独立开发者亨德里克·曼斯（Hendrik Mans）主导开发，以技术严谨性和架构清晰性著称。Chatto 的核心设计选择极具辨识度：采用 NATS 作为消息传输骨干，利用兼容 S3 的对象存储保存媒体文件，并为每位用户单独管理加密密钥，确保数据主权。与追求极致规模化的主流聊天工具不同，Chatto 坚持简洁性、可审计性与部署自主性——它以一个轻量、自包含的二进制文件形式交付，仅需极少基础设施即可运行。

该平台的设计回应了市场对去中心化、透明通信工具日益增长的需求。它选用 NATS——一种用 Go 编写的轻量级云原生消息系统——实现了低延迟的发布/订阅语义，同时避免了 Kafka 或 RabbitMQ 的复杂性。NATS 内置流持久化引擎，意味着消息历史可直接在 NATS 中可靠存储，无需额外数据库；也可选择将数据卸载至外部兼容 S3 的存储（如 MinIO 或 AWS S3）。这种深度集成大幅降低了运维复杂度，又不失扩展弹性。

Chatto 在技术上的另一大亮点是其开发方式：全程采用“智能体编程”（agentic coding）——一种新兴范式，即在人类主导下，由 AI 智能体协助完成代码编写、测试与优化。正如社区同行所指出，这并非简单自动化，而是一种协作式工程实践：AI 负责生成样板代码、编写测试用例、提出架构改进建议，而开发者始终保有最终审查权与决策权。成果体现为功能完备且文档详实，官方文档（docs.chatto.run）提供了清晰的部署与运维指南。

隐私与合规被深度融入设计：用户完全掌控自身密钥；账户删除时，关联密钥将被密码学意义上的彻底擦除——尽管这一设计也引发了关于企业适用性的讨论（例如法律要求的数据保留可能需软删除机制）。目前尚不支持原生 Slack 或 Discord 互操作，但已提供 Slack 到 Chatto 的迁移工具，表明项目正朝渐进式生态整合方向演进。对于重视通信基础设施自主权、警惕厂商锁定或监管风险的团队而言，Chatto 是一个成熟、可立即投入生产的有力选项。
