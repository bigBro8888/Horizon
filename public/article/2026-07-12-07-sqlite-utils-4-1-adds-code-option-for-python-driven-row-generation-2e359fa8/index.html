sqlite-utils 4.1 新增 --code 选项，支持内联 Python 生成数据行

新闻时间: 2026-07-12T04:35:50.468606+00:00 · 来源: rss

sqlite-utils 4.1 是 Simon Willison 开发的 SQLite 命令行与 Python 工具库的最新小版本更新，距离 4.0 版发布仅数日。本次更新引入多项实用改进，其中最引人注目的是新增 --code 选项，它允许开发者直接在命令行中嵌入 Python 代码来动态生成待插入或更新的数据行，彻底摆脱对 JSON、CSV 等外部文件的依赖。用户不再需要预先准备静态数据文件，而是可将逻辑内联书写——无论是简短的代码块，还是独立的 .py 脚本——只要定义一个 rows() 生成器函数或 rows 可迭代对象即可。

该选项同时支持 sqlite-utils insert 和 sqlite-utils upsert 命令。例如，一条命令即可通过 yield 返回两个含 ID 与名称的字典，全程无需文件 I/O，也无需序列化/反序列化开销。这种设计延续了 sqlite-utils 的核心理念：将数据首先视为 Python 对象，再映射为数据库记录。它并非孤立创新，而是对已有能力的自然延伸——此前工具已支持 --convert 进行列级 Python 转换、--functions 注入自定义 SQL 函数；--code 则将“可执行逻辑”从数据处理层提升至数据生成层。

除 --code 外，4.1 版还包含多项开发者友好的增强：新增 --type column-name type 参数，可强制覆盖 SQLite 自动推断的列类型（例如将形如 '02134' 的 ZIP 码明确设为 TEXT，避免丢失前导零）；新增 sqlite-utils drop-index 命令及 ignore=True 容错机制；sqlite-utils query 现支持从标准输入读取 SQL（用 - 占位），完美契合 Unix 管道哲学。这些改进共同强化了 sqlite-utils 作为轻量级、可组合、终端原生数据工具的定位。
