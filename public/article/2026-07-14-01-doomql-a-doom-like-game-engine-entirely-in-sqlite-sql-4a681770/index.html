DOOMQL：完全基于SQLite纯SQL实现的类Doom游戏引擎

新闻时间: 2026-07-14T04:15:20.855259+00:00 · 来源: rss

DOOMQL 不仅仅是一款终端游戏——它彻底重构了数据库的潜在能力。该项目由 Peter Gostev 开发（使用了实验性工具链，包括 GPT-5.6 Sol），将 SQLite 从被动的数据存储器转变为活跃的运行时游戏引擎，驱动一款完整可玩的第一人称复古射击游戏。每一帧画面——角色移动、碰撞检测、敌人AI、武器开火、生命值管理，甚至逐像素的屏幕渲染——都不是由 C 或 Python 代码执行，而是通过精心编排的 SQL 查询在 SQLite 内部实时计算完成。最终呈现的是一款令人惊叹的 ASCII 风格像素化 Doom 克隆，在单个 SQLite 数据库文件中全栈运行。

其技术核心是一个递归公用表表达式（CTE），实现了完整的光线投射（raycasting）渲染器——这一技术传统上仅见于 C 等编译型语言。该纯 SQL 渲染器能计算墙面距离、应用透视投影、通过查找表实现纹理映射，并合成精灵图（sprites），全程无需外部库或过程式逻辑。游戏主循环以约每秒 30 帧的速度运行，由一个轻量级 Python 主机脚本驱动：它轮询键盘输入并触发 SQL 执行；而所有可变状态——玩家坐标、弹药数量、敌人位置，甚至当前帧缓冲区——均以 (x, y, r, g, b) 列形式存于 frame_pixels 视图中，全部驻留在数据库内。

DOOMQL 的真正魅力在于其高度可观察性与可组合性。由于全部状态都存于 SQLite，开发者可借助 Datasette 等工具实时查询与可视化。配合新推出的 Datasette Apps 插件，用户能构建动态网页仪表盘——例如自动刷新的 3D 视角视图或战术俯视小地图——这些应用以声明式 HTML/JS 编写，直接向同一数据库发起 SQL 查询。有用户甚至仅用自然语言提示（向 Claude Fable 5 发送指令）就生成了视角视图和小地图，充分展现了 SQL、UI 与游戏逻辑前所未有的深度统一。

DOOMQL 并非面向生产环境的高性能引擎——它是一个概念验证项目，旨在突破 SQL 的传统边界。但其影响深远：它挑战了“计算应发生在何处”的固有认知，为调试复杂状态机提供了全新范式，并有力证明——即使是在终端中，富有表现力的声明式逻辑也能驱动丰富交互体验。
