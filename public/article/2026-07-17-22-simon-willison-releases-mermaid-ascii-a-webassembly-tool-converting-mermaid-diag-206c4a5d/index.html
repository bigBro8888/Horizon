西蒙·威利森发布 mermaid-ascii：基于 WebAssembly 的 Mermaid 图表转彩色 ASCII 艺术工具

新闻时间: 2026-07-17T04:21:50.862828+00:00 · 来源: rss

西蒙·威利森发布了名为 mermaid-ascii 的新型浏览器端工具，可将 Mermaid 图表源码实时转换为富含色彩的 ASCII 艺术图——全部在客户端通过 WebAssembly 运行。与传统终端渲染器不同，该工具能忠实保留 Mermaid 语法中的语义样式（例如颜色类定义），并将其渲染为浏览器中支持 ANSI 色彩的文本，特别适用于文档预览、CLI 风格的说明生成，或在 Slack、邮件、README 等纯文本优先环境中轻量共享图表。

该工具基于 Alexander Grooff 开发的开源 Go 库（github.com/AlexanderGrooff/mermaid-ascii），该库已实现完整的 Mermaid 解析器与 ASCII 渲染器。威利森并未用 Rust 或 JavaScript 重写逻辑，而是借助 Anthropic 推出的 Mythos 级编码大模型 Claude Fable 5，自动调用 TinyGo 将原有 Go 代码编译为 WebAssembly。这一实践凸显了开发者工具链的新范式：大语言模型不仅用于生成代码，更可用于跨语言编译协调与遗留代码再利用。

其核心亮点在于完整支持色彩渲染。如在线演示所示，Mermaid 中的 classDef good color:#3fb950 等类定义被精准转换为终端兼容的 ANSI 转义序列，生成绿色、黄色和红色文字区块，并支持灵活的间距控制（X/Y 方向填充、方框内边距）及多行标签处理。界面提供“仅 ASCII”开关、复制纯文本与生成可分享永久链接等功能，强化了其在协作场景中的低摩擦实用性。

此次发布紧随威利森此前推出的 grok-mermaid 工具之后，后者基于 xai-org/grok-build 的 Rust 代码编译为 WebAssembly。通过对比 Rust 与 Go 两种独立实现，他印证了 WebAssembly 如何赋能多语言工具复用——无需牺牲性能或保真度。尽管二者均不替代 SVG 或 PNG 等出版级图像格式，却精准填补了一类独特需求：离线可用、终端完美缩放、零依赖的可视化方案，天然适配文本主导的技术生态。
