Gleam 编程语言正式入驻去中心化代码托管平台 Tangled

新闻时间: 2026-07-19T04:32:10.428248+00:00 · 来源: hackernews

Gleam — 一种为 Erlang 虚拟机（BEAM）构建的静态类型函数式编程语言 — 已正式入驻去中心化开源代码托管与协作平台 Tangled。该公告发布于 tangled.org/gleam.run/gleam，其意义远不止于新增一个镜像站点：它体现了 Gleam “小而审慎”的设计哲学与 Tangled 所倡导的用户数据主权、分布式开发工作流之间的主动契合。

与 GitHub 或 GitLab 等中心化代码平台不同，Tangled 采用点对点架构：开发者可自行运行名为“Knot”（绳结）的节点来托管仓库，并通过“Spindle”（纺锤）在本地机器上执行持续集成。这种模式强调数据自主权、规避厂商锁定——这与 Gleam 社区一贯推崇的极简主义和工具链克制高度共鸣。Gleam 自带编译器、格式化器、包管理器及编辑器插件，整套工具链高度内聚且立场鲜明；此次将 Tangled 列为官方托管选项，正是这一理念向基础设施层的自然延伸。

然而，集成过程并非一帆风顺。早期用户反馈了若干实际障碍：登录流程需手动查收邮件验证，还需学习一套陌生的社交化 URL 协议；本地 Knot 与公共 Tangled 界面之间常出现仓库更新不同步；尤其在纯 IPv6 网络环境下，必须额外配置 IPv4 NAT 和伪造 DNS A 记录才能稳定运行。这些痛点揭示了去中心化技术落地时的真实挑战，即便面对的是经验丰富的开发者群体。

此举也引发更深层的价值观讨论。有社区成员质疑：为何 Gleam 选择由风险资本支持、2025 年成立的初创公司 Tangled，而非 Codeberg 等成熟、非营利的开源替代方案？另一些人则盛赞其象征意义：两者均以清晰性、可组合性与反冗余为信条。正如一位曾在 Ubuntu 峰会结识 Gleam 团队的开发者所言，其演讲全程手绘动画，每一帧都在无声强化“小而审慎”的设计信念——如今，这一信念正从语言本身延展至基础设施的选择。
