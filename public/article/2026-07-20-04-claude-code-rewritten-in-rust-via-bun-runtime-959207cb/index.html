Claude Code 重写为基于 Rust 的 Bun 运行时

新闻时间: 2026-07-20T04:49:41.561405+00:00 · 来源: hackernews

Anthropic 已悄然但坚定地重构了其面向开发者的 AI 工具 Claude Code 的技术底座——将其从原先基于 Zig 的自定义实现，全面迁移至 Bun 这一由 Rust 编写的高性能 JavaScript 运行时。这并非简单的依赖升级或版本更新，而是一次彻底的架构重写，其核心驱动力是内存安全、行为确定性与长期可维护性等深层工程诉求。据 Bun 创始人 Jarred Sumner 及 Simon Willison 在 2026 年 7 月的分析，原始 Zig 代码要求开发者手动管理内存生命周期——这种繁琐且易出错的方式，在并发场景及 AI 代理驱动的工作负载下，持续引发一系列隐蔽的内存错误。

迁移到 Bun 带来的不仅是 Rust 的所有权模型；更关键的是，它将整个技术栈统一于一个轻量、可嵌入、零依赖的单一可执行文件之上。Bun 内置打包器、转译器与包管理器，全部针对速度与正确性深度优化，并采用 Safari 的 JavaScriptCore 引擎（而非 Node.js 或 Deno 所用的 V8），从而在 macOS 和 iOS 工具链中实现更紧密集成。对 Anthropic 而言，这意味着更快的迭代周期、终端 UI 渲染中更少的段错误，以及与现有 TypeScript 前端工具链（用于 Claude Code 的 TUI 层）更强的兼容性。

尤为关键的是，这一转变折射出行业更广泛的范式迁移：尽管 Zig 曾因提供类 C 的控制力与更低学习门槛吸引系统工程师，但 2026 年数据显示，Rust 正在 AI 基础设施领域加速普及——其编译期内存安全保证能彻底消除整类内存相关故障，而这在自主编码代理频繁操作抽象语法树（AST）与文件系统的场景中尤为关键。此次重构也表明 Anthropic 日益倚重垂直整合的运行时：Bun 已不再仅是 Node.js 的替代品，而是 AI 原生开发工具的战略性底层基座。

当然，这一过渡并非毫无争议。社区讨论呈现明显分歧——部分开发者盛赞其安全性提升与行为确定性增强；另一些人则批评决策过程不透明、单次合并逾百万行代码且耗时不足一月，以及将完整 JS 运行时嵌入本可原生构建的二进制程序所引发的哲学性质疑。然而，其技术动因依然坚实：当人类开发者与 AI 代理都具有非确定性时，运行时本身必须成为可靠性的终极来源。
