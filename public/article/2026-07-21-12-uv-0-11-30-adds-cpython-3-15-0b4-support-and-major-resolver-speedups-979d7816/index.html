uv 0.11.30 支持 Python 3.15b4 并显著提升依赖解析速度

新闻时间: 2026-07-21T04:28:45.909412+00:00 · 来源: github

uv 团队发布了 0.11.30 版本，这是一次目标明确的补丁更新，为 Python 开发者带来了切实可用的改进——尤其利好正在试用新语言版本或维护大型单体仓库（monorepo）的用户。该版本于 2026 年 7 月 20 日发布，正式支持 CPython 3.15.0b4，即 Python 3.15 的最终测试版，确保早期使用者能无缝使用 uv 高速安装与依赖解析能力，无需担心兼容性断层。

除 Python 版本支持外，本次更新还着力解决实际工作流中的痛点。工作区（workspace）用户将获得更智能的元数据同步体验：uv workspace metadata --sync 新增 --active 参数，可直接作用于当前激活的虚拟环境；当通过符号链接访问工作区时，uv 现在会复用已有的集中式项目环境，避免重复创建和磁盘空间浪费。这些改进让 uv 在 Python 单体仓库等复杂协作场景中更加可靠。

性能提升是本次发布的最大亮点。依赖解析器现在会跳过所有文件均被 exclude-newer 排除的候选包；限制并行缓存读取以降低调度开销；借助 toml_writer 加速 lockfile 序列化；压缩缓存中的 Simple API 分发元数据与哈希值；将陈旧缓存条目的解码合并为单个阻塞任务；并将载荷解码移出解析器工作线程；同时缓存 Python 环境标记（requirement markers）并在多次解析中复用。这些优化共同降低了内存压力、CPU 竞争和 I/O 延迟，尤其在对大型项目反复执行 uv sync 或 uv lock 时效果明显。

修复项同样务实：tar-wheel 卸载逻辑不再误删无关文件；Unix 系统下的 pyvenv.cfg 文件现在能准确保留字面量 extends-environment 路径，提升了基于 venv 工具链的可重现性。文档方面新增了贡献指南，进一步降低社区参与门槛。尽管属于小版本迭代，0.11.30 充分体现了 uv 工程团队的开发哲学：专注正确性、易用性与可测量的速度提升，不盲目扩张功能，也不牺牲稳定性。
