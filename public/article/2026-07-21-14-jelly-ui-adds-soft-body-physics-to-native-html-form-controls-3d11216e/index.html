Jelly UI为原生HTML表单控件添加果冻式物理效果

新闻时间: 2026-07-21T04:28:45.283690+00:00 · 来源: hackernews

Jelly UI 是一个无需依赖的 Web Components 库，它将软体物理模拟叠加到标准 HTML 表单元素（如按钮、复选框、单选框和滑块）上，使这些控件在交互时像果冻一样形变、回弹。与 CSS 过渡或简单缓动函数不同，Jelly UI 基于 requestAnimationFrame 构建了自定义物理引擎，实时模拟质量-弹簧系统动力学，赋予每个控件真实的触觉反馈。

该库的魅力在于其视觉新颖性和即时表现力：按钮晃动、滑块拉伸、复选框颤动，整体呈现出有机、可触摸的材质感。它采用 MIT 开源协议，不依赖任何框架，直接增强原生 <button> 和 <input> 元素，支持通过 <jelly-button> 标签或 jelly 属性启用。演示网站展示了多种常见表单场景下的效果，并声明在系统启用 prefers-reduced-motion 时能自动降级为静态样式。

但技术审查也揭示了明显权衡。有开发者发现，Jelly UI 每 8 毫秒就运行一次全局 RAF 循环——无论页面中是否有活跃的果冻组件——导致整页强制重绘，可能引发卡顿，尤其在低端设备或多组件页面上。此外，交互逻辑偏离原生行为：鼠标按住按钮后拖出再释放仍会触发点击，但同样操作在复选框上却不会响应——这一细微差异影响了可用性与无障碍兼容性。

社区讨论还延伸至更深层的设计价值观。有人赞赏其怀旧趣味（联想到 2000 年代 Flash 时代的实验界面），也有人指出当代 UI 趋向克制、清晰与性能优先。虽然 Jelly UI 尊重系统级减少动画偏好，却未提供页面内开关，削弱用户自主控制权。目前，它处于前端实验艺术与实用界面工程的交汇点——富有启发性、技术精巧，但尚不适合多数对性能或无障碍要求严格的生产环境。
