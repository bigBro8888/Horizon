uv 0.11.32 发布：新增包检查、锁文件标准化与工作区元数据

新闻时间: 2026-07-24T04:29:06.009474+00:00 · 来源: github

Python 包管理工具 uv 正式发布 0.11.32 版本，带来多项面向可复现性、可维护性和开发者体验的实用改进。该版本于 2026 年 7 月 23 日发布，新增了两个命令：支持按包名筛选的 uv check 和能同时更新多个环境标记（marker）下同一包版本的 uv upgrade；同时强化了锁文件（lockfile）校验机制，并支持自动格式修复。

最值得关注的变化在于锁文件处理逻辑。从本版本起，uv lock --check 及所有启用 --locked 参数的命令，将拒绝加载非标准格式的锁文件——即未严格遵循 uv 定义的字段顺序、缩进和规范化结构的文件。若检测到此类文件，开发者可直接运行 uv lock --refresh 自动重生成符合规范的版本。这一调整呼应了 Python 社区近期的重大进展：2025 年 4 月正式通过的 PEP 751，确立了统一的 pylock.toml 锁文件标准。

工作区（workspace）功能也得到增强：uv workspace metadata 命令现在默认包含当前 Python 环境的尽力而为信息，例如解释器路径与版本号，便于开发者和 CI 系统快速掌握执行上下文。此外，一项底层性能优化跳过了无法推导新冲突时的冗余依赖组冲突展开步骤，显著加快大型或复杂依赖图的解析速度。

修复方面，一个关键问题是当 Requires-Python 兼容性声明仅在分发包元数据中动态发现（而非提前声明）时，uv 会正确分叉解析路径，避免错误兼容判断。所有预编译二进制包均覆盖九大平台，包括 Apple Silicon 与 Intel macOS、ARM64 与 x86 Windows、RISC-V 与 PowerPC Linux 等，体现其坚定的跨平台支持。安装方式保持简洁：一行 Shell 或 PowerShell 脚本即可完成，也可直接下载带 SHA256 校验码的压缩包。
