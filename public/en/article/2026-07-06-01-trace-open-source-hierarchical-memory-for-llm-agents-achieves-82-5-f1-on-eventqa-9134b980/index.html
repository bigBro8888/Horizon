TRACE: Open-source hierarchical memory for LLM agents achieves 82.5% F1 on EventQA

News time: 2026-07-06T14:35:56+00:00 · Source: reddit

A new open-source memory architecture called TRACE has emerged as a compelling alternative to traditional flat retrieval-augmented generation (RAG) systems for large language model (LLM) agents. Unlike conventional approaches that store and retrieve conversation history as unstructured or chunked text, TRACE organizes memory hierarchically—building a dynamic topic tree where each branch represents a coherent theme, annotated with summaries and temporal links. This design draws inspiration from cognitive theories like multiple trace theory, which posits that memories are encoded as unique, attribute-rich neural traces rather than static copies.

The system was rigorously evaluated on MemoryAgentBench’s EventQA task—a benchmark introduced at ICLR 2026 designed to test an agent’s ability to track and reason over temporal event chains in narrative texts. EventQA follows an 'inject once, query multiple times' paradigm: a single long-form story is ingested incrementally across turns, then probed with multiple questions about what happens next. This setup stresses memory retention, coherence tracking, and causal inference—not just keyword matching.

Using the open-weight GPT-OSS-20B model (a 21B-parameter MoE model with 3.6B active parameters), TRACE achieved an 82.5% F1 score—significantly outperforming established baselines: Mem0 scored 37.5% and MemGPT/Letta 26.2%, both using GPT-4o-mini under official reported conditions. Notably, these comparisons aren’t strictly apples-to-apples due to model differences—but the gap remains striking even when accounting for architectural disparities. The authors openly acknowledge limitations, including parsing failures when adapting Mem0 to GPT-OSS-20B (a known issue with JSON strictness across many open models) and Letta’s server dependency, which prevented direct comparison.

TRACE is distributed as a PyPI package (pip install trace-memory) and includes full JSON logs of all runs for reproducibility. Its modular design allows integration into existing agent frameworks, and its topic-tree structure enables intuitive debugging, pruning, and user-guided memory navigation. While scalability and optimal tree depth control remain open questions—topics actively discussed in the Reddit thread—the project sets a new bar for transparent, open-weight memory evaluation in agentic AI.
