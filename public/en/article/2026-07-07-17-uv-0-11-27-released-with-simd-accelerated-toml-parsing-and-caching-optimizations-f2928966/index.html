uv 0.11.27 released with SIMD-accelerated TOML parsing and caching optimizations

News time: 2026-07-07T02:27:41.921923+00:00 · Source: github

The Astral team has released uv version 0.11.27 — a focused maintenance update that delivers tangible performance gains across core operations while refining usability for Python developers. Released on July 6, 2026, this patch builds on uv’s reputation as an ultra-fast, Rust-powered alternative to pip and virtualenv, emphasizing speed, correctness, and modern tooling ergonomics.

At the heart of this release is SIMD-accelerated TOML parsing — a low-level optimization that leverages modern CPU instruction sets (like AVX-512 or ARM NEON) to parse pyproject.toml and uv.lock files significantly faster. This change, implemented via optimized parsing routines, reduces overhead during project initialization and lockfile resolution — especially impactful in large monorepos or CI environments where TOML I/O is frequent. Complementing this are multiple caching improvements: default dependency markers are now cached when reading lockfiles, wheel metadata fetching continues past ignored errors, and --python-downloads-json-url responses are now cached — all reducing redundant network and filesystem calls.

Usability sees subtle but meaningful upgrades too. The uv workspace list --scripts command now discovers extensionless shebang scripts (e.g., #!/usr/bin/env python), aligning better with Unix conventions and improving script discoverability in Cargo-style workspaces. Error handling has also been hardened: uv now gracefully reports missing versions in registry-backed uv.lock entries instead of panicking, and blank lines in uv pip tree output for empty dependency trees have been eliminated. Several memory and allocation optimizations — such as reusing package names from wheel filenames and reading cache entries into exact-sized buffers — further shrink runtime overhead.

Beyond performance and stability, the release includes infrastructure updates: the Rust toolchain has been upgraded to 1.96.1, and public APIs previously removed were restored to support downstream tools like Pixi. While no major new features ship in 0.11.27, its cumulative impact is substantial — users upgrading from earlier versions will notice snappier lock resolution, faster installs, and more predictable behavior in complex workspace setups. For teams adopting uv at scale, this release represents steady, reliable progress toward a more robust and performant Python toolchain.
