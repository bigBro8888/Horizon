uv 0.11.28 released with security-hardened ZIP parsing and GraalPy 25.1.3

News time: 2026-07-08T04:29:38.425107+00:00 · Source: github

The Rust-based Python tooling project uv has released version 0.11.28, marking a significant step in both security hardening and developer experience refinement. Released on July 7, 2026, this patch delivers critical updates to how uv handles ZIP archives—long a high-risk surface for supply chain and dependency injection attacks—by upgrading its underlying ZIP library, astral-async-zip, to v0.0.20. This update implements 15 targeted changes designed to eliminate parser differentials: subtle inconsistencies where different components (e.g., a downloader vs. an installer) interpret the same ZIP file differently, potentially enabling malicious payloads to bypass validation.

Beyond security, the release upgrades the embedded GraalPy runtime—the high-performance, JVM-hosted Python 3 implementation—to version 25.1.3. GraalPy is increasingly relevant for data science and ML workflows due to its JIT compilation and strong compatibility with PyTorch and SciPy; this update ensures uv’s tooling remains aligned with the latest GraalVM ecosystem improvements. The upgrade also reflects uv’s broader strategy of supporting multiple Python runtimes beyond CPython, enhancing portability and performance in polyglot environments.

Developer-facing enhancements focus heavily on clarity and consistency. Error reporting has been overhauled: multiline error causes now preserve indentation, full cause chains are rendered by default, and all errors—including those from uv build and tool requirements—are formatted uniformly and respect quiet flags (-q, -qq). Guidance around lockfile updates has been moved from warnings to contextual hints, reducing noise while maintaining usability. These changes collectively reduce cognitive load during debugging and CI troubleshooting.

Performance optimizations are pervasive but low-level—22 distinct memory allocation avoidance patches across resolver logic, URL handling, platform detection, and metadata formatting. While individually minor, they compound into measurable latency reductions, especially in large dependency graphs or constrained environments like CI runners. Notably, bytecode compilation is now limited only to installed distributions during uv pip install, avoiding unnecessary work when dependencies are merely resolved or cached. Together, these updates reinforce uv’s identity as a fast, secure, and ergonomic alternative to traditional Python tooling.
