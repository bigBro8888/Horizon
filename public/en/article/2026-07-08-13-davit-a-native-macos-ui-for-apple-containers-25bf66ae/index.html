Davit: A Native macOS UI for Apple Containers

News time: 2026-07-08T04:29:37.868101+00:00 · Source: hackernews

Davit is a newly launched, open-source macOS application that delivers a polished, Apple-style graphical interface for managing containers—built not on Electron or web views, but natively in Swift. Designed as a developer-first tool, it interfaces directly with Apple’s ContainerAPIClient library via XPC, the same inter-process communication mechanism used by Apple’s official container CLI. This architectural choice eliminates reliance on shell subprocesses or external binaries, enabling fast, secure, and responsive operations like listing containers, streaming logs, pulling images, managing volumes and networks, executing commands inside containers (davit exec), and even bootstrapping launchd services—all from a sleek, intuitive UI.

Unlike many cross-platform container tools, Davit embraces macOS conventions: it’s signed and notarized by Apple, weighs just 17 MB, and installs required runtime components (like the container platform daemon) automatically on first launch. Its rapid development—28 commits in three days, totaling over 5,000 lines of Swift—reflects a focused, pragmatic engineering effort. Notably, every commit credits Claude Fable 5 as co-author, signaling AI-assisted but human-guided development—a pattern increasingly associated with high-quality native tooling.

User feedback has been overwhelmingly positive. Early adopters report seamless setup—e.g., downloading the runtime and successfully running nginx:latest on first try—and praise its native feel, Docker compatibility, and security-conscious architecture. Comparisons to Orbstack highlight Davit’s ambition: while Orbstack excels in performance and integration, Davit offers an open, lightweight, and fully native alternative rooted in Apple’s own container stack. Some users suggest practical enhancements, such as adding a step-by-step tutorial or exploring VM-based agent isolation to keep LLM runtimes strictly off the host.

Davit doesn’t replace Docker, nor does it aim to compete with cloud-native container orchestration. Instead, it fills a growing niche: local, macOS-native container management that feels like a first-party Apple app—fast, secure, ergonomic, and built for developers who value both power and polish.
