Chatto, a self-hostable NATS-based chat platform, is now open source

News time: 2026-07-09T05:04:10.099604+00:00 · Source: hackernews

Chatto — a privacy-first, self-hostable real-time chat platform — has officially entered the open-source ecosystem. Developed by Hendrik Mans, a respected independent developer known for technical rigor and clarity, Chatto distinguishes itself through deliberate architectural choices: it uses NATS as its core messaging backbone, leverages S3-compatible object storage for media persistence, and implements per-user cryptographic key management to ensure data sovereignty. Unlike mainstream chat tools built for scale-at-all-costs, Chatto prioritizes simplicity, auditability, and deployment control — shipping as a compact, self-contained binary that runs with minimal infrastructure overhead.

The platform’s design reflects a growing demand for alternatives to centralized, opaque communication tools. By embracing NATS — a lightweight, cloud-native messaging system written in Go — Chatto achieves low-latency pub/sub semantics without the complexity of Kafka or RabbitMQ. Its built-in stream persistence engine means operators don’t need separate databases for message history; instead, messages are durably stored within NATS itself or optionally offloaded to external S3-compatible storage (e.g., MinIO or AWS S3). This tight integration reduces operational surface area while preserving flexibility.

What sets Chatto apart technically is its development methodology: it was built using agentic coding — an emerging paradigm where AI agents assist in writing, testing, and refining code under human direction. As noted by peers in the community, this isn’t just automation — it’s a collaborative engineering process where agents handle boilerplate, generate tests, and propose architecture refinements, while the developer retains full ownership and review authority. The result is both highly functional and unusually well-documented, with clear operational guidance available at docs.chatto.run.

Privacy and compliance are baked into the model: users retain full control over their keys, and account deletion triggers cryptographic shredding of associated keys — though this has sparked discussion about enterprise applicability, where soft-delete policies may be required for legal retention. While native Slack or Discord interoperability isn’t yet supported, a Slack-to-Chatto migration tool exists, signaling intent toward gradual ecosystem integration. For teams seeking autonomy over communication infrastructure — especially those wary of vendor lock-in or regulatory exposure — Chatto represents a compelling, production-ready starting point.
