Developer Runs GLM-5.2 on 32GB RAM Laptop Using Int4 Quantization and On-Demand Weight Streaming

News time: 2026-07-10T05:05:40.268474+00:00 · Source: hackernews

A solo developer named JustVugg has achieved a notable milestone in local LLM deployment: running GLM-5.2 — a state-of-the-art, 744-billion-parameter Mixture-of-Experts (MoE) model — on a consumer-grade laptop with just 32GB of RAM. What makes this especially impressive is that GLM-5.2 normally demands high-end GPU infrastructure; yet here, there’s no GPU at all. Instead, the project — named Colibrì — relies entirely on CPU inference, aggressive int4 quantization, and a novel memory-aware architecture that streams only the active expert weights from disk as needed.

The core insight behind Colibrì lies in how it handles GLM-5.2’s MoE structure. While the full model contains 744B parameters, only about 40B are activated per token — and among those, just ~11GB of routed expert weights change dynamically across layers. Colibrì exploits this sparsity by keeping the dense components (attention layers, shared experts, embeddings — ~17B params) permanently resident in RAM at int4 precision (~9.9 GB), while storing the 21,504 individual routed experts (~370 GB total at int4) on SSD. These experts are loaded on-demand using an LRU cache, optionally pinned hot storage, and the OS page cache as a free second-level buffer.

Remarkably, the entire inference engine fits in a single ~1,300-line C file (c/glm.c) — no Python, no BLAS, no CUDA, no GPU drivers. It was written and tested end-to-end on a 12-core laptop with only 25GB RAM — meaning every optimization decision was validated under real hardware constraints. Though cold-start latency remains high and generation speed modest (~0.1 tokens/second), the priority wasn’t speed but feasibility: proving that cutting-edge open models can run locally without specialized hardware.

This effort resonates deeply within the local-inference community, where accessibility and hardware democratization are central goals. Unlike cloud-dependent or GPU-bound alternatives, Colibrì reclaims agency for developers with mid-tier machines — turning theoretical ‘possible-in-principle’ into tangible, reproducible reality. Its minimalist design also invites scrutiny, extension, and collaboration: several commenters have already shared complementary approaches, from Metal-accelerated macOS variants to llama.cpp integrations and unified-memory streaming strategies.
