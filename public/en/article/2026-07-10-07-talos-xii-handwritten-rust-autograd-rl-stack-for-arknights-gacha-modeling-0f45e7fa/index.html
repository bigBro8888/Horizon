Talos-XII: Handwritten Rust Autograd + RL Stack for Arknights Gacha Modeling

News time: 2026-07-10T05:05:39.994125+00:00 · Source: reddit

Talos-XII is a remarkable solo engineering project: a framework-free, command-line gacha simulator for Arknights: Endfield, built entirely in Rust from the ground up. Rather than relying on static probability tables — which only answer 'what’s the chance of getting X in N pulls?' — Talos-XII models player decision-making under uncertainty using custom-trained neural networks and reinforcement learning agents. It tackles strategic questions that matter to real players: 'Can I afford the rate-up operator using only free currency?', or 'Given my current pity count, should I pull now or save for next banner?' These aren’t statistical trivia — they’re resource-allocation decisions shaped by stochastic environments, time horizons, and behavioral constraints.

The system comprises four tightly integrated models, all trained on first run (30–45 seconds) and cached for reuse. An 'EnvNet' MLP learns environment noise — simulating subtle, unobserved biases in the game’s RNG. A 'Luck Optimizer' operates over a 32-dimensional engineered feature space capturing pity progress, streaks, and interaction effects. A Dueling DQN handles discrete 'pull vs. wait' choices, while a PPO actor-critic with a Multi-Head Latent Attention (MLA) transformer models continuous strategy — like pacing pulls across multiple banners. Crucially, none of this depends on PyTorch, tch-rs, or ndarray; every tensor operation, gradient computation, and optimization step is hand-implemented.

Under the hood lies a meticulously crafted autograd engine supporting matmul, conv2d, pooling, and normalization — all with verified backward passes. It features runtime SIMD dispatch (scalar → AVX2 → AVX-512 → NEON), Rayon-parallelized simulation (10,000+ sims/sec on a laptop), BF16 inference caching, and optional PyO3 bindings for Python-based training scripting. The codebase is rigorously tested (142 unit tests), CI-verified across Linux/Windows/macOS including ARM64 cross-compilation, and ships as a single MIT-licensed static binary.

A particularly experimental component is ACHF (Adaptive Cache-aware Hyper-Connections): a novel layer blending dense and pruned sparse paths via a gradient-sensitive gate, applying Sinkhorn-manifold weight projection, and dynamically switching execution modes based on measured latency. Unlike large-scale hypernetworks, ACHF targets compact CPU-bound RL policies inside a self-contained binary — making it an unusual testbed for hardware-aware model design. Its efficacy remains unproven beyond the author’s own machine, hence the open call for benchmarking across diverse hardware — especially AVX-512, ARM64 NEON, and GPU setups — where negative results are explicitly welcomed as valuable data.
