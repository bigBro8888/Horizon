PostgreSQL Reimplemented in Rust Passes Full Regression Test Suite

News time: 2026-07-10T05:05:40.268628+00:00 · Source: hackernews

A solo developer named Malisper has released pgrust — a ground-up reimplementation of PostgreSQL written entirely in Rust — that now passes 100% of PostgreSQL’s official regression test suite. This milestone, verified across more than 46,000 SQL queries, signals functional parity with PostgreSQL 18.3 on core SQL semantics and behavior. The project isn’t just a technical curiosity: it includes a live WebAssembly demo hosted at pgrust.com, allowing users to run real SQL queries directly in their browser without any backend server. This demonstrates not only correctness but also portability and embeddability — traits rarely seen in traditional database systems.

What makes pgrust especially notable is its development methodology. The author explicitly leverages large language models (LLMs) as co-pilots throughout the rewrite process — from generating initial scaffolding to translating complex C logic into idiomatic Rust, and even assisting with query planner design. Rather than treating LLMs as black-box code generators, the approach treats them as accelerants for architectural refactoring: applying modern systems engineering principles — memory safety, async-first concurrency, modular design — to a codebase originally shaped by decades of incremental evolution.

Still, pgrust remains experimental. It does not yet support all PostgreSQL features — notably missing are advanced replication protocols, logical decoding, and many extensions — and lacks production-hardened components like WAL recovery or crash-safe durability guarantees. Its current architecture prioritizes correctness and test coverage over performance optimization; benchmarks show it runs significantly slower than upstream PostgreSQL on equivalent workloads. Yet the project’s existence challenges assumptions about how legacy systems can be modernized: not through slow, conservative patching, but via deliberate, AI-augmented rearchitecture.

The broader implications extend beyond databases. If a single developer can replicate PostgreSQL’s SQL semantics with full test compliance in under a month — aided by LLMs and Rust’s tooling — it suggests a new paradigm for maintaining critical infrastructure: one where domain knowledge is codified, validated, and iterated upon rapidly, rather than ossified in aging C codebases. Whether pgrust evolves into a viable alternative or serves as a blueprint for future rewrites, it underscores a pivotal shift — from rewriting despite complexity to rewriting because of it.
