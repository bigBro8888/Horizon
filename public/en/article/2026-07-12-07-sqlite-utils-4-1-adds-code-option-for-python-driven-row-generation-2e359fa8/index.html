sqlite-utils 4.1 Adds --code Option for Python-Driven Row Generation

News time: 2026-07-12T04:35:50.468606+00:00 · Source: rss

sqlite-utils 4.1 — the latest minor release of Simon Willison’s popular CLI and Python library for SQLite database manipulation — introduces several pragmatic enhancements, with the --code option standing out as the most expressive new feature. Released just days after version 4.0, this update empowers developers to define rows programmatically directly from the command line, eliminating the need to pre-stage data in JSON, CSV, or other file formats. Instead of importing static files, users can now embed Python logic — either as an inline code block or via a .py file — that yields dictionaries representing rows to insert or upsert into a table.

The --code flag works with both sqlite-utils insert and sqlite-utils upsert, accepting either a string of Python code or a path to a script. Crucially, it expects either a rows() generator function or a rows iterable — a design consistent with sqlite-utils’ existing philosophy of treating data as Python objects first, database records second. For example, a simple invocation might yield two pet names with IDs, all defined in one compact, readable block — no external files, no intermediate serialization, and full access to Python’s expressiveness (e.g., loops, conditionals, imports, or API calls).

This feature builds naturally on prior capabilities: sqlite-utils already supported inline Python for column transformations via --convert, and even allowed custom SQL functions via --functions. The --code option extends that same principle — executable, declarative logic at the CLI level — to row generation, not just row transformation. It reflects a broader trend in modern CLI tooling: blurring the line between scripting and data operations, enabling lightweight ETL-like workflows without leaving the terminal.

Beyond --code, version 4.1 delivers several other quality-of-life improvements. A long-requested --type column-name type option lets users override SQLite’s automatic type inference — critical for preserving leading zeros in ZIP codes or enforcing TEXT over INTEGER where semantics matter. New commands like sqlite-utils drop-index and support for reading SQL queries from stdin (sqlite-utils query db.db -) further cement sqlite-utils’ role as a versatile, Unix-philosophy-aligned toolkit — composable, scriptable, and respectful of pipelines and developer intent.
