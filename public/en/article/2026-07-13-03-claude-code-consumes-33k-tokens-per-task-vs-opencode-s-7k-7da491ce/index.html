Claude Code consumes 33k tokens per task vs OpenCode’s 7k

News time: 2026-07-13T04:43:03.431769+00:00 · Source: hackernews

A recent empirical study by Systima.ai has exposed a striking disparity in token efficiency between two popular agentic coding assistants: Anthropic’s Claude Code and the open-source OpenCode. While both tools aim to automate software development tasks through autonomous planning and execution, the analysis reveals that Claude Code routinely consumes approximately 33,000 tokens per coding task—nearly five times more than OpenCode’s ~7,000 tokens under comparable conditions. This finding emerged not from synthetic benchmarks, but from real-world instrumentation: researchers inserted logging middleware between each agent and its LLM endpoint, capturing every API request and its associated token usage metadata—including input, output, and tool-calling overhead.

The inefficiency appears rooted in Claude Code’s architectural choices—notably its aggressive use of sub-agents and redundant tool orchestration. As one user recounted, a single complex request triggered seven concurrent sub-agents, exhausting budget limits before any meaningful output was generated. In contrast, sequential execution by a single agent succeeded without issue. Further, even trivial prompts like 'Hey' or 'commit' sometimes provoked over 30 tool invocations in Claude Code, suggesting systemic token inflation rather than isolated edge cases. The authors note this isn’t merely about prompt size—it reflects how the agent harness interprets intent, selects tools, caches context, and manages internal state.

Beyond raw numbers, the study raises deeper questions about design philosophy and economic incentives. Critics speculate that high token consumption may align with Anthropic’s subscription-based revenue model—especially given restrictions preventing users from routing Claude Code’s sub-agents through alternative coding backends. Meanwhile, OpenCode’s leaner architecture, built for local or self-hosted deployment, prioritizes minimalism and transparency: its full system prompt is publicly viewable on GitHub. The community response has been robust, with over 280 technical comments debating measurement validity, advocating for qualitative outcome comparisons, and urging reproducibility—prompting Systima to commit to publishing full input-output traces and expanded task evaluations.

Ultimately, this isn’t just about cost—it’s about sustainability, accessibility, and trust in AI-assisted development. As agentic coding tools become embedded in daily workflows, inefficient token usage translates directly into slower iteration, higher cloud bills, and reduced adoption among resource-constrained teams. The comparison underscores a growing tension in the AI ecosystem: autonomy versus efficiency, convenience versus control, and closed design versus open accountability.
