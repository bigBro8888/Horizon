shot-scraper 1.11 Improves Server Startup Reliability and CLI Consistency

News time: 2026-07-13T04:43:02.962448+00:00 · Source: rss

Simon Willison has released shot-scraper 1.11, a focused maintenance update that strengthens the tool’s robustness in real-world automation workflows. The release centers on two practical improvements: significantly extending the timeout for server startup during multi-page and video capture sessions, and standardizing JavaScript injection options across commands. These changes reflect a maturing CLI tool that prioritizes reliability and developer ergonomics over flashy new features.

The most impactful change addresses a long-standing pain point for users running shot-scraper in server mode — particularly with multi and video commands. Previously, the tool waited only one second for a locally launched web server to become responsive before failing. In practice, especially on slower machines or when serving complex static sites, this often led to brittle failures. Version 1.11 replaces that rigid delay with an adaptive polling mechanism that waits up to 30 seconds, checking port availability repeatedly until the server responds or the timeout expires. This makes automated screenshot and video generation far more resilient in CI environments or resource-constrained setups.

Equally valuable is the introduction of the --js-file option across five core commands (shot-scraper, pdf, html, accessibility, and har). Instead of embedding JavaScript logic directly via --javascript, users can now reference external files — including local paths, stdin (via -), or even GitHub-hosted scripts using the gh:username/script shorthand. This supports better version control, reuse, and readability for non-trivial browser-side logic. The multi command also gains corresponding YAML support via the js_file: key, enabling declarative configuration for complex multi-step captures.

Additional polish includes adding a --timeout flag to shot-scraper javascript and html commands, bringing them in line with other subcommands. While none of these changes constitute a paradigm shift, together they signal thoughtful evolution: reducing friction, preventing avoidable failures, and aligning interfaces across the tool’s growing feature set. For documentation maintainers, QA engineers, and developers building automated visual regression or demo pipelines, shot-scraper 1.11 delivers quiet but meaningful reliability gains.
