Mindgard Discloses Unpatched Cursor 0day Allowing Arbitrary Code Execution

News time: 2026-07-15T04:15:47.835753+00:00 · Source: hackernews

In a stark demonstration of the growing tension between security researchers and AI-powered tool vendors, Mindgard has publicly disclosed a six-month-old zero-day vulnerability in Cursor — the popular AI-augmented integrated development environment (IDE). First identified on December 15, 2025, the flaw allows arbitrary code execution when Cursor encounters a maliciously named git.exe binary placed directly in a project directory. Despite repeated reporting through HackerOne — including initial rejection as 'Informative' and 'out of scope' — the issue remained unpatched across 197+ Cursor releases, persisting even in the latest version tested by Mindgard.

The vulnerability hinges on Cursor’s internal reliance on Git binaries for repository operations. When Cursor scans a local project folder and detects an executable named git.exe, it executes it without validation or sandboxing — a design choice that assumes trust in the local filesystem. This bypasses standard OS-level protections and enables privilege escalation if the binary is weaponized. Notably, no user interaction beyond opening the affected repository is required; Cursor automatically triggers the binary during routine Git integration checks.

Mindgard’s disclosure comes amid mounting criticism of Cursor’s security posture and vendor responsiveness. As highlighted in recent analyses, Cursor operates not as a passive editor but as a constellation of high-autonomy agents — each representing a potential attack surface. Its security model emphasizes Privacy Mode to prevent data leakage to LLM providers, yet offers no equivalent safeguards against local binary execution. The absence of input validation, path sanitization, or execution sandboxing for Git-related binaries reveals a critical gap in threat modeling.

The broader implications extend beyond Cursor itself. With over 1 million monthly active users and deep integration into developer workflows — including CI/CD pipelines and agent-assisted coding — this vulnerability underscores how AI IDEs inherit and amplify traditional software supply chain risks. It also reignites the long-standing debate between responsible disclosure and full disclosure: when vendors fail to act despite clear evidence and repeated engagement, does public disclosure become an ethical necessity? For many developers now relying on AI agents to write, test, and deploy code, the answer may no longer be theoretical.
