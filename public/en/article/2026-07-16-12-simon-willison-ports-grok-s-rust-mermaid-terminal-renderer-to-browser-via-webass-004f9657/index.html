Simon Willison ports Grok’s Rust Mermaid terminal renderer to browser via WebAssembly

News time: 2026-07-16T04:23:13.221851+00:00 · Source: rss

Developer Simon Willison has built a novel browser-based tool called grok-mermaid that renders Mermaid diagram syntax into Unicode box art — all running client-side with no server dependency. The tool is powered by WebAssembly and directly leverages the open-sourced mermaid.rs renderer from xAI’s Grok CLI, a self-contained Rust module originally designed for terminal output. Willison discovered this compact, dependency-free implementation while exploring Grok Build’s recently released codebase — specifically the xai-grok-markdown/src/mermaid.rs file — and recognized its potential for lightweight, portable rendering beyond the command line.

Rather than reimplementing the logic, Willison used AI-assisted development (Claude Code / Fable 5) to guide the WebAssembly porting process. The resulting tool accepts Mermaid flowchart, sequence, or graph TD code in a text editor and instantly renders it as ASCII/Unicode box art — not SVG or canvas, but pure text using Unicode block characters and box-drawing glyphs. This approach yields minimal bundle size, zero external dependencies, and full offline capability. A live demo shows a clean UI with controls for max width, copying rendered output as plain text, and generating shareable permalinks.

The choice of Unicode box art over vector graphics reflects a deliberate trade-off: sacrificing visual fidelity and interactivity for extreme portability and accessibility. Unlike traditional Mermaid renderers that rely on JavaScript-heavy SVG generation or DOM manipulation, grok-mermaid compiles Rust to WebAssembly and executes entirely within the browser’s sandboxed runtime. It avoids bundling large JS libraries like D3.js or Mermaid.js, making it ideal for constrained environments — documentation previews, CLI-integrated web dashboards, or even terminals embedded in web apps.

This project exemplifies how modern tooling bridges ecosystems: a terminal-first Rust library, open-sourced by an AI company (xAI), is repurposed via WebAssembly to serve web developers — without requiring changes to the original code. It also highlights the growing role of AI pair programming in systems-level integration work: Willison openly credits Claude Code for scaffolding the WASM glue code, demonstrating how LLMs can accelerate cross-platform porting when domain knowledge (Rust, WASM, Mermaid AST) is combined with precise prompting. Ultimately, grok-mermaid isn’t just a novelty — it’s a reproducible blueprint for bringing performant, native-like CLI tools into the browser.
