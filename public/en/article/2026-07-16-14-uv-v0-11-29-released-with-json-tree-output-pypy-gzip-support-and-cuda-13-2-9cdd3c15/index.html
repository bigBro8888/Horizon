uv v0.11.29 released with JSON tree output, PyPy gzip support, and CUDA 13.2

News time: 2026-07-16T04:23:14.166821+00:00 · Source: github

Astral has released uv version 0.11.29 — a focused maintenance update that refines usability, security, and ecosystem compatibility without altering core architecture. Released on July 15, 2026, this iteration delivers tangible improvements for Python developers who rely on uv as a high-performance alternative to pip, pip-tools, and virtualenv. The release emphasizes developer ergonomics: uv tree now supports machine-readable JSON output, enabling seamless integration with IDEs, CI pipelines, and dependency visualization tools. Meanwhile, PyPy downloads are now served with gzip compression, reducing bandwidth usage and accelerating installation times — especially valuable in CI/CD environments or low-bandwidth scenarios.

Under the hood, uv expands its hardware acceleration readiness by adding official support for CUDA 13.2 in PyTorch installations — a timely update given NVIDIA’s recent toolchain stabilization and growing adoption of CUDA 13.x in ML workflows. The resolver also receives subtle but impactful tweaks: diagnostics for unsatisfiable version constraints are now clearer, missing extras errors now name the affected project explicitly, and conflicting root requirements no longer show empty version ranges — all contributing to faster debugging cycles. Performance gains come from smarter caching reuse across commands like uv tree, uv export, and uv audit, plus deferred setup for no-op uv sync operations.

Security hardening is another highlight: uv now rejects PEP 517 backend paths that escape the source tree — including symlink-based escapes — and redacts credentials from failed Git fetch logs. It also improves OSV vulnerability auditing by splitting large queries (>1,000 packages) and applying fixed-version fixes only to matching ecosystems, preventing false positives. Bug fixes address edge cases like versionless .egg-info parsing, post-release range ordering (now fully PEP 440 compliant), and idempotent locking behavior for complex cross-platform constraints.

While not a feature milestone, v0.11.29 exemplifies uv’s mature development rhythm: incremental, well-tested, and user-centric. With over 10,000 PyPI packages validated in its test suite and Tier 1 support across macOS (Apple Silicon/x86_64), Linux, and Windows, uv continues consolidating its role as the de facto Rust-powered foundation for modern Python tooling — bridging the gap between speed, safety, and simplicity.
