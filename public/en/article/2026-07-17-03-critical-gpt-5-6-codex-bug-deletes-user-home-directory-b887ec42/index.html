Critical GPT-5.6 Codex Bug Deletes User Home Directory

News time: 2026-07-17T04:21:50.862779+00:00 · Source: rss

A newly disclosed safety-critical bug in OpenAI’s GPT-5.6-powered Codex coding agent has raised urgent concerns among developers and AI safety researchers. According to Thibault Sottiaux — a senior engineer involved in Codex’s reliability testing — the flaw triggers catastrophic file deletion when users enable ‘full access mode’ without sandboxing or auto-review safeguards. In such configurations, the agent attempts to override the $HOME environment variable to set up a temporary working directory. However, due to a logic error in how the model interprets or executes shell commands, it mistakenly treats the overridden $HOME as a target for recursive deletion — resulting in the complete erasure of the user’s home directory.

This is not theoretical: multiple real-world reports have confirmed unintended deletions across Linux and macOS systems where Codex was run locally with elevated permissions. The issue does not occur in Codex’s default cloud-based execution mode — which runs inside isolated containers with internet disabled and strict filesystem boundaries — but emerges specifically when users opt into local, unsandboxed execution for greater control or offline capability. Full access mode, intended for advanced automation tasks like project-wide refactoring or dependency updates, thus becomes a vector for irreversible data loss if basic safety layers are disabled.

What makes this especially alarming is its root cause: an environment-variable manipulation gone wrong, rather than a classic code injection or privilege escalation. Unlike traditional software bugs, this reflects a deeper challenge in autonomous agent design — where LLMs reason about system state using natural language, then translate that reasoning into unsafe shell operations without robust validation. The absence of automatic review or sandboxing transforms a well-intentioned temporary-directory setup into a destructive command. Developers relying on Codex for local development workflows — particularly those integrating it into CI/CD pipelines or IDE extensions — must now re-evaluate their trust assumptions.

OpenAI has acknowledged the issue internally and is reportedly prioritizing a patch that enforces mandatory sandboxing or auto-review when $HOME-related environment overrides are detected. Until then, best practice strongly advises against enabling full access mode outside strictly controlled, disposable environments — and never on production or personal machines. This incident underscores a broader truth in the era of agentic AI: granting agents direct system access demands not just permission models, but semantic guardrails that understand intent, context, and consequence — not just syntax.
