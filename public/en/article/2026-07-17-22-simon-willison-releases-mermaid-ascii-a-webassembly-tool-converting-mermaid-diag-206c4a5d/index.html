Simon Willison releases mermaid-ascii: a WebAssembly tool converting Mermaid diagrams to colored ASCII art

News time: 2026-07-17T04:21:50.862828+00:00 · Source: rss

Simon Willison has launched mermaid-ascii, a novel browser-based tool that transforms Mermaid diagram source code into rich, color-aware ASCII art — all running entirely client-side via WebAssembly. Unlike traditional terminal renderers, this tool preserves semantic styling (e.g., class definitions for colors) from Mermaid syntax and renders them as ANSI-colored text in the browser, making it uniquely suited for documentation previews, CLI-style documentation generation, or lightweight diagram sharing in text-first environments like Slack, email, or READMEs.

The tool builds on an existing open-source Go library by Alexander Grooff (github.com/AlexanderGrooff/mermaid-ascii), which implements a comprehensive Mermaid parser and ASCII renderer. Rather than rewriting the logic in Rust or JavaScript, Willison leveraged Claude Fable 5 — Anthropic’s Mythos-class coding model — to automate the compilation of that Go code to WebAssembly using TinyGo. This approach highlights a pragmatic shift in developer tooling: using LLMs not just for code generation, but for cross-compilation orchestration and legacy code repurposing.

A key differentiator is full color support. As demonstrated in the live demo, Mermaid class definitions like classDef good color:#3fb950 are faithfully translated into terminal-compatible ANSI escape sequences, rendering green, yellow, and red text blocks with proper spacing, padding controls (X/Y/padding), and multi-line label handling. The UI includes intuitive toggles for ASCII-only output and options to copy both plain text and shareable permalinks — reinforcing its utility for collaborative, low-friction workflows.

This release follows Willison’s earlier grok-mermaid tool, which used Rust code from xai-org/grok-build compiled to WebAssembly. By comparing two independently developed implementations — one Rust-based, one Go-based — he underscores how WebAssembly enables polyglot tool reuse without sacrificing performance or fidelity. While neither tool replaces SVG or PNG rendering for publication-quality output, they fill a distinct niche: fast, embeddable, zero-dependency visualizations that work offline, scale perfectly in terminals, and integrate seamlessly into text-based ecosystems.
