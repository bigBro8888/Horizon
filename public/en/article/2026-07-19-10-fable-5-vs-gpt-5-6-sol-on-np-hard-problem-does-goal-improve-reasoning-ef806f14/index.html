Fable 5 vs. GPT-5.6 Sol on NP-hard problem: does /goal improve reasoning?

News time: 2026-07-19T04:32:10.428222+00:00 · Source: hackernews

A recent empirical evaluation by Charles Azam pits Anthropic’s Claude Fable 5 against OpenAI’s newly released GPT-5.6 Sol on a carefully constructed NP-hard combinatorial optimization problem — one where brute-force search is infeasible and success hinges on strategic reasoning, memory retention, and goal-directed exploration. The core question isn’t just which model solves more instances, but whether the /goal prompt directive — a lightweight instruction that explicitly anchors the model’s objective at the start of a session — meaningfully shifts performance, especially under constrained or iterative search conditions.

The test setup deliberately avoids trivial benchmarks: it uses a custom NP-hard variant inspired by constraint satisfaction and graph partitioning, requiring models to balance feasibility, optimality, and stepwise justification. Both models were evaluated across identical prompts, with and without /goal, while controlling for token budget, temperature, and output format. Notably, the comparison focuses on reasoning architecture rather than raw accuracy — probing how each system navigates search spaces, avoids local optima, and sustains task focus over multi-turn interactions.

Results show nuanced trade-offs. GPT-5.6 Sol — billed by OpenAI as its ‘workhorse’ coding and reasoning model — demonstrates stronger baseline performance in single-step, high-precision solving. Fable 5, meanwhile, shows greater sensitivity to /goal: its success rate improves significantly when the directive is present, suggesting better alignment between instruction anchoring and internal planning mechanisms. However, this advantage diminishes in longer sessions, hinting at underlying memory limitations — a point echoed by community commenters who report Fable 5 (and Claude models generally) struggling to retain critical context over extended workflows.

Beyond head-to-head scores, the post sparks deeper reflection on agentic design. As one commenter notes, /goal appears most effective for ‘single-track investigations’ — linear, bounded tasks — whereas more complex problems may benefit from structured parallelism (e.g., Ultra mode’s adversarial review and checkpointed branching). This implies that prompt directives like /goal are not universal fixes but contextual levers, whose efficacy depends on problem topology, model memory fidelity, and the presence of complementary search scaffolding. Ultimately, the experiment underscores a growing consensus: evaluating frontier LLMs requires moving beyond pass/fail metrics toward fine-grained analysis of how reasoning unfolds — and where it breaks down.
