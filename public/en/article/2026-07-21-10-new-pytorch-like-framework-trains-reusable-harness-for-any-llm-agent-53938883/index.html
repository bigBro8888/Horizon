New PyTorch-like framework trains reusable 'harness' for any LLM agent

News time: 2026-07-21T04:28:45.374189+00:00 · Source: reddit

A new open-source framework called harness-training reframes how we improve AI agents—not by fine-tuning large language models (LLMs), but by training a separate, frozen controller layer called a 'harness'. This harness sits between the task environment and any LLM, guiding its behavior through structured prompts, tool routing, memory handling, and step-by-step observation formatting. Crucially, once trained against one LLM and one task benchmark—like SWE-Bench—the harness remains fixed and can be plugged into any other LLM (e.g., Claude, GPT-4, or local models) and evaluated on entirely different tasks, such as Terminal-Bench. The result is transferable capability gains without retraining the underlying model.

The framework draws inspiration from PyTorch’s design philosophy: modular, composable, and developer-friendly. It supports OpenAI-compatible APIs out of the box and integrates with evaluation benchmarks like Terminal-Bench and SWE-Bench. Users define training logic via configuration files and Python objects—such as AgenticEstimator for environment interaction and CodexAgentBackend for LLM interfacing. Training proceeds in epochs where each iteration compares a candidate harness update against the current baseline using a StrictPareto criterion: an update is only accepted if it improves all measured metrics (e.g., success rate, cost, latency) simultaneously—no trade-offs allowed.

Optimization uses a custom GreedyMonotonic algorithm that incrementally advances the harness state only when monotonic improvement is guaranteed—akin to version-controlled 'git commits' for agent behavior. Each accepted change becomes the new baseline; rejected ones are preserved as references. This ensures deterministic, auditable progress—a lesson learned after early versions suffered from non-reproducible outcomes due to stochastic prompting and environment noise.

In practice, the harness trained on SWE-Bench coding tasks improved performance on Terminal-Bench system navigation tasks—even though those environments differ significantly in interface, tools, and reward structure. The author reports beating Terminal Bench 2.0’s baseline (Terminus Harness) and demonstrates cross-task generalization previously seen only in expensive full-model fine-tuning. As the GitHub repo shows, this isn’t just conceptual—it’s engineered for real-world extensibility, with clear abstractions for backends, estimators, criteria, and optimizers. For developers building agentic systems, harness training offers a path toward reusable, model-agnostic intelligence upgrades.
