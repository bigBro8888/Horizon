uv 0.11.30 adds CPython 3.15.0b4 support and major resolver speedups

News time: 2026-07-21T04:28:45.909412+00:00 · Source: github

The uv team has released version 0.11.30, a tightly focused patch that delivers tangible improvements for Python developers—especially those working with upcoming language versions or large monorepos. Released on July 20, 2026, this update extends official support to CPython 3.15.0b4, the final beta release of Python 3.15, ensuring early adopters can use uv’s fast installation and resolution workflows without compatibility gaps.

Beyond Python version support, the release targets real-world pain points in dependency management. Workspace users benefit from smarter metadata synchronization: uv workspace metadata --sync now accepts --active to target the current virtual environment directly, and symlinked workspace access now reuses centralized project environments—reducing redundant setup and disk usage. These changes make uv more robust for complex, shared-development setups like Python monorepos.

Performance gains are the standout feature. The resolver now skips candidate packages whose files are entirely excluded by exclude-newer, cuts parallel cache reads to lower scheduling overhead, and serializes lockfiles faster using toml_writer. Cached metadata is compacted, stale cache entries are decoded in bulk (not per-worker), and Python requirement markers are cached across resolver runs. Collectively, these optimizations reduce memory pressure, CPU contention, and I/O latency—especially noticeable during repeated uv sync or uv lock operations on large projects.

Bug fixes round out the release: tar-wheel uninstall logic no longer accidentally deletes unrelated files, and Unix pyvenv.cfg now preserves literal extends-environment paths—improving reproducibility for venv-based tooling. Documentation also improves with a new contribution guide, lowering the barrier for community involvement. While incremental, 0.11.30 reflects uv’s disciplined engineering rhythm: prioritizing correctness, ergonomics, and measurable speed—without bloating the CLI surface or compromising stability.
