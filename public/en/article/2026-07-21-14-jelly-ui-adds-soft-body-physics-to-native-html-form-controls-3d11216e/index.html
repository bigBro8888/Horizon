Jelly UI adds soft-body physics to native HTML form controls

News time: 2026-07-21T04:28:45.283690+00:00 · Source: hackernews

Jelly UI is a new, dependency-free Web Components library that overlays soft-body physics simulations onto standard HTML form elements—buttons, checkboxes, radio inputs, and sliders—making them visually deform and rebound like gelatinous objects when interacted with. Unlike CSS transitions or simple easing functions, Jelly UI uses a custom physics engine built on requestAnimationFrame (RAF) to model mass-spring dynamics in real time, giving each control a tactile, expressive response to clicks, hovers, and drags.

The library’s appeal lies in its novelty and aesthetic immediacy: buttons wobble, sliders stretch, and checkboxes jiggle—evoking the playful, organic feel of physical materials. It’s MIT-licensed, open source, and designed to work without frameworks, directly enhancing native <button> and <input> elements via custom tags like <jelly-button> or attributes like jelly. Its demo site showcases this behavior across common form patterns, all while claiming graceful degradation for users who enable prefers-reduced-motion.

Yet technical scrutiny reveals trade-offs. A deep dive by a developer found that Jelly UI runs a global RAF loop every 8ms—regardless of whether components are active—triggering full-document repaints. This can cause measurable jank, especially on lower-end devices or pages with many jelly-enabled elements. Further, interaction logic diverges from native expectations: holding and dragging away before releasing registers as a click on buttons but not on checkboxes—a subtle but meaningful inconsistency for usability and accessibility.

Critics also raise broader design questions. Some praise its nostalgic charm—comparing it to early Flash-era experiments—but others note how generational shifts in UX sensibility have moved toward restraint, clarity, and performance-first principles. The library does respect reduced-motion preferences at the OS level, but lacks an in-page toggle, limiting user agency. For now, Jelly UI sits at the intersection of experimental frontend art and practical interface engineering—provocative, technically clever, but not yet production-ready for most accessibility- or performance-sensitive applications.
