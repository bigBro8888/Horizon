Jack Dorsey launches open-source Buzz workspace with chat, Git, and AI agents

News time: 2026-07-22T04:31:11.845342+00:00 · Source: hackernews

Jack Dorsey — co-founder of Twitter and Block — has unveiled Buzz, an open-source, self-hosted collaboration platform designed for software teams. Unlike mainstream tools like Slack or GitHub, Buzz unifies real-time team chat, version-controlled code hosting (Git), and AI-powered agents into a single, decentralized workspace. Its core innovation lies in how it structures data: all interactions — messages, code commits, agent actions — are encoded as signed Nostr events, cryptographically tied to user identities and relayed through the Nostr protocol’s decentralized network.

Buzz is built from the ground up for data sovereignty. Teams can host it entirely on their own infrastructure, avoiding cloud vendor lock-in and retaining full control over sensitive code, conversations, and AI behavior. This aligns with Dorsey’s long-standing advocacy for decentralized protocols and user-owned data — a philosophy previously reflected in initiatives like Bluesky and Nostr itself. The platform does not rely on centralized servers for core functionality; instead, it leverages Nostr relays for event distribution while keeping private state (like Git repositories or agent memory) local or on trusted infrastructure.

The integration of AI agents is particularly distinctive. Rather than treating agents as isolated assistants, Buzz embeds them as first-class participants in team workflows — they can read chat threads, inspect Git diffs, propose PRs, and even collaborate with humans using emoji-rich, context-aware dialogue. However, this ‘multiplayer agent’ model raises architectural questions: as one commenter noted, granting agents broad visibility across team channels creates privacy and access-control challenges that demand fine-grained permission systems — unlike simpler ‘single-player’ agents that act only on behalf of one user.

Early reactions reflect both enthusiasm and healthy skepticism. Some developers applaud the ambition to rethink collaboration beyond SaaS silos, while others question whether Nostr — optimized for lightweight social messaging — scales reliably for enterprise-grade Git hosting and synchronous team coordination. UX concerns have also surfaced, including playful naming conventions (e.g., ‘@Honeybot’) and interface aesthetics that clash with engineering pragmatism. Still, Buzz signals a meaningful shift: developer tooling is no longer just about better UIs or faster builds — it’s increasingly about who controls the data, who trains the agents, and how trust is enforced at the protocol layer.
