Claude Tag handles 65% of Claude Code team’s PRs internally

News time: 2026-07-22T04:31:11.207512+00:00 · Source: rss

At the AI Engineer World's Fair in July 2026, Simon Willison hosted a fireside chat with Cat Wu and Thariq Shihipar from Anthropic’s Claude Code team — offering rare, candid insight into how the company builds and deploys its own AI coding tools. The discussion covered Claude Code, the newly launched Claude Tag Slack integration, the autonomous coding agent Fable, security practices, evaluation frameworks, and tool design philosophy. Crucially, it revealed not just what Anthropic ships, but how and why: through rigorous internal dogfooding — or, as they call it, "ant fooding" — and retention-driven shipping decisions.

One standout metric was Claude Tag’s rapid internal adoption: it now handles 65% of product engineering pull requests for the Claude Code team itself. This isn’t experimental usage — it’s operational reality. The team relies on Claude Tag to triage, draft, and even land PRs asynchronously in Slack channels, where it operates as a shared, always-on teammate under its own identity. Meanwhile, Claude Code follows a strict internal-first rollout policy: features ship externally only after demonstrating sustained user retention among Anthropic engineers — a deliberate filter against feature bloat and low-value additions.

The conversation also highlighted major shifts in prompt engineering practice. For newer models like Fable 5 and Opus 4.8, Anthropic has moved away from verbose system prompts filled with examples and prohibitions. Their Claude Code system prompt recently shrank by 80%, and they’ve found that lists of "don’t do X" instructions actually degrade output quality. Instead, they rely more on model-native reasoning capabilities and structured auto-mode configurations — a key enabler for Claude Tag’s reliability.

Thariq offered a philosophical note on human–agent collaboration: rather than resisting the cognitive shift caused by powerful coding agents (a phenomenon he likened to the "Deep Blue effect"), engineers should respond by taking on more ambitious projects — leveraging agents to handle implementation so humans can focus on architecture, UX, and problem framing. And Fable’s capabilities extend beyond code: Thariq used it to edit the launch video for Fable itself, underscoring its multimodal competence. Ultimately, Anthropic’s success hinges not just on technical capability, but on cultural habits — like working in public Slack channels — that make agent integration visible, auditable, and iterative.
