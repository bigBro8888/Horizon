Software factories fail because AI can’t grasp human intent

News time: 2026-07-24T04:29:05.208170+00:00 · Source: hackernews

A new critical analysis titled Why Software Factories Fail (or: harness engineering is not enough) argues that the core bottleneck in AI-powered software factories isn’t technical execution — it’s the persistent inability of coding agents to reliably capture, interpret, or evolve human intent. The article challenges the common assumption that better tooling, faster LLMs, or improved prompt engineering will eventually solve the problem. Instead, it positions intent alignment — the faithful translation of a developer’s mental model, business context, and long-term vision into code — as a fundamentally human task that resists automation.

The piece draws on real-world experience, citing a July 2025 ‘lights-off’ experiment where teams attempted fully autonomous agent-driven development. While commenters debate whether model capabilities meaningfully shifted around fall 2025–spring 2026, the article maintains that even state-of-the-art agents remain brittle when faced with ambiguous requirements, shifting priorities, or undocumented architectural assumptions. One commenter frames this as the ‘Intent-Implement-Quality’ problem: agents excel at implementation given a crisp one-liner spec, but that spec itself is a compressed, lossy artifact of human reasoning — and no current system can reverse-engineer the full intent behind it.

Another key insight centers on codebase understanding. As one engineer observes, ‘Claude can write the code for you but it can’t understand it for you.’ Agents may generate syntactically correct, even functionally adequate code — but they lack the lived, contextual knowledge of how modules interconnect, why certain patterns were chosen, or what trade-offs were made years ago. That understanding evolves slowly, through conversation, documentation, and shared memory — not token context windows.

Finally, the article highlights PR review as a systemic chokepoint. Even when agents produce clean diffs, human reviewers still bear the cognitive load of verifying alignment with unstated goals, security constraints, and team norms. Tools like GitHub’s PR interface compound the friction — making review less about insight and more about endurance. Ultimately, the argument isn’t anti-AI; it’s pro-realism: scaling software delivery requires augmenting human judgment, not replacing it.
