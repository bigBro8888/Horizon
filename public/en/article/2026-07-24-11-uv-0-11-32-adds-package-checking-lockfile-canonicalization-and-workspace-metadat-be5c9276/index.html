uv 0.11.32 adds package checking, lockfile canonicalization, and workspace metadata

News time: 2026-07-24T04:29:06.009474+00:00 · Source: github

The Python packaging tool uv has released version 0.11.32, delivering several practical enhancements aimed at improving reproducibility, maintainability, and developer ergonomics. Released on July 23, 2026, this patch introduces two new CLI commands — uv check with selective package targeting and uv upgrade with support for multi-marker updates — alongside stricter validation and automatic correction of lockfile formatting.

One of the most notable changes is how uv now handles lockfiles. Starting with this release, uv lock --check and any command using --locked will reject lockfiles that aren’t canonically formatted — meaning they don’t follow the standardized ordering, indentation, and field layout defined by uv’s internal lockfile specification. When such non-canonical files are detected, uv lock --refresh can automatically regenerate them in the correct format. This aligns with broader industry efforts toward lockfile standardization, including Python’s recently ratified PEP 751 and its pylock.toml format.

The update also improves workspace support: uv workspace metadata now includes best-effort details about the active Python environment — like interpreter path and version — by default. This helps developers and CI systems better understand context without manual probing. Meanwhile, a subtle but impactful performance optimization skips unnecessary conflict expansion during dependency resolution when no new conflicts can be inferred — speeding up large or complex resolves.

Bug fixes round out the release, notably one that correctly forks universal resolutions when Requires-Python constraints are only discovered from distribution metadata (not declared upfront), preventing incorrect compatibility assumptions. All binaries are available across nine platforms — from Apple Silicon and x86 macOS to ARM64 Windows, RISC-V Linux, and PowerPC — reflecting uv’s strong cross-platform commitment. Installation remains frictionless via shell or PowerShell scripts, or direct binary download with SHA256 checksums for verification.
